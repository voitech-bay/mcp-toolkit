# Сценарии использования (как этим пользоваться)

Этот документ — инструкция “что нажимать и в каком порядке”, чтобы получать данные, аналитику, обогащение (enrichment) и быстро готовить ответы.

---

## Быстрый ориентир: куда заходить

- **`/sync`**: синк данных из GetSales + синк **Analytics snapshots** (метрики по дням)
- **`/conversations`**: диалоги LinkedIn, фильтры по статусу ответа, запуск “Start reply”
- **`/hypotheses`**: гипотезы и их компании
- **`/companies`**, **`/contacts`**: ручная работа со справочником (поиск, привязки)
- **`/enrichment`**: очередь и результаты агентов (company/contact), управление промптами
- **Redash**: дашборды и графики по итоговым таблицам и снапшотам

---

## Сценарий 1: Базовая подготовка проекта (один раз на проект)

**Цель:** чтобы кнопка синка работала и данные/графики появлялись автоматически.

- Выбираешь проект (в шапке приложения).
- Заходишь в **`/sync` → API Credentials**.
- Вставляешь **Base URL** и **API Key** для источника (GetSales).
- Нажимаешь **Save Credentials**.

Что должно получиться:
- В проекте появляется “Key set”.
- Можно запускать синк без ручных настроек окружения.

---

## Сценарий 2: Регулярный синк данных (операционка)

**Когда:** ежедневно/несколько раз в неделю — чтобы новые контакты, сообщения и кампании были в базе.

- Открываешь **`/sync`**.
- Нажимаешь **Start Sync**.
- Ждёшь завершения (видишь лог в реальном времени).

После синка обновятся:
- контакты и их профили
- сообщения/переписки
- отправители (senders)
- кампании (flows) и участники кампаний (flow leads)

---

## Сценарий 3: Синк аналитики (снапшоты по дням для графиков)

**Когда:** перед еженедельным отчётом или когда нужно сравнение по неделям.

Важно: это **отдельно** от “Start Sync”. Тут подтягиваются “снимки метрик” по дням.

- Открываешь **`/sync` → Analytics snapshots**.
- Выбираешь **Date range** (например “Last 7 days”).
- Нажимаешь **Sync analytics**.

Что происходит:
- дни, которые уже есть в базе, будут пропущены
- после этого в Redash корректно строятся тренды/воронка по датам

---

## Сценарий 4: Ежедневная работа с ответами (inbox → быстрый reply)

**Цель:** находить диалоги, понимать контекст и быстро готовить варианты ответа.

- Открываешь **`/conversations`**.
- Выбираешь фильтр статуса:
  - **Got response** — у кого уже есть входящий ответ (обычно приоритет)
  - **Waiting for response** — кому мы написали последними (ждём ответ)
  - **No response** — кому ответов не было
- Кликаешь на диалог слева → справа видишь тред сообщений и карточку контакта.

Если у контакта нет компании в базе:
- нажимаешь **Attach company from DB**
- выбираешь/прикрепляешь компанию

Если нужно сразу разложить контакт по гипотезам:
- в блоке **Hypotheses** добавляешь компанию в гипотезу (или убираешь)

---

## Сценарий 5: “Start reply” через Cursor (контекст → 3 варианта ответа)

**Цель:** получить качественные варианты ответа с учётом компании, гипотезы и истории переписки.

- В диалоге на странице **`/conversations`** нажимаешь **Start reply conversation**.
- В модалке выбираешь, что включить в контекст (обычно: компания + контакт + текущая переписка; иногда добавляешь гипотезу и related contacts).
- Нажимаешь сборку контекста (в приложении создаётся “снимок контекста”).
- Открываешь Cursor по ссылке и просишь сгенерировать ответ — Cursor подтягивает сохранённый контекст и предлагает варианты.

Рекомендованный ритм:
- берёшь **Variant 1–3** → выбираешь лучший → слегка правишь руками → отправляешь уже “человеческий” финальный текст.

---

## Сценарий 6: Управление гипотезами (что тестируем и на ком)

**Цель:** держать структуру “гипотеза → список компаний”, чтобы аналитика и команда говорили на одном языке.

- Открываешь **`/hypotheses`**.
- Создаёшь гипотезу (название + описание + target persona).
- Добавляешь в гипотезу компании (через UI гипотез или из карточки диалога/компании).

Как использовать дальше:
- смотришь в Redash по гипотезам: кто даёт ответы/конверсии, где просадка, что отключать/усиливать

---

## Сценарий 7: Enrichment (обогащение компаний/контактов агентами)

**Цель:** быстро заполнить “карточку смысла” и дополнительные поля для персонализации и анализа.

- Открываешь **`/enrichment`**.
- Выбираешь вкладку:
  - **Company enrichment** — когда работаешь по аккаунтам/компаниям
  - **Contact enrichment** — когда важнее персональный контекст
- (Опционально) фильтруешь по **contact list** — чтобы обрабатывать конкретный сегмент.

Запуск агентов:
- выбираешь агент в выпадающем списке → выделяешь строки → **Enqueue selected**
- или запускаешь агент по одной строке (кнопка Run/Rerun в ячейке)
- или запускаешь “Run all on page” у колонки агента

Что дальше:
- ячейки меняют статус: **Queued → Agent working → Result / Error**
- по клику на **Result** смотришь, что агент вернул
- по клику на **Error** читаешь причину и запускаешь **Rerun**

## Сценарий 8: Настройка агентов и промптов (качество enrichment)

**Цель:** чтобы агенты выдавали нужный формат и полезные поля под ваш ICP.

Управление агентами:
- **`/enrichment` → Manage agents**
- создаёшь агента или редактируешь существующего (name, entity type, prompt, batch size, active)

Редактирование промпта “по месту”:
- в заголовке колонки агента нажимаешь иконку редактирования (карандаш)
- включаешь **Preview**, чтобы увидеть, как промпт будет выглядеть на реальных строках таблицы
- используешь подсказки переменных вида `{{...}}` (автодополнение)

Подсказка:
- если batch size > 1, агент будет работать “пакетом” (удобно для дешёвого/быстрого массового enrichment)

---

## Сценарий 9: Мониторинг воркеров и “почему долго”

**Цель:** понимать, выполняются ли задачи и не застряло ли что-то.

В **`/enrichment`** смотри:
- индикатор **Live**
- “Worker batch …” — сколько задач воркеры держат в буфере перед запуском batch
- batch-id чипы в ячейках → можно открыть **Batch detail** и посмотреть детали

Если что-то зависло:
- открой список runs/jobs (`/enrichment/jobs`) и найди ошибки
- конкретные задачи можно остановить/перезапустить (через UI, если вынесено, или через сервисные ручки)

---

## Рекомендованный “ритм” по неделе (практика)

- **Каждый день:** `Conversations → Got response → Start reply → отправка`
- **2–3 раза в неделю:** `Sync → Start Sync`
- **Раз в неделю (перед отчётом):** `Sync → Analytics snapshots (Last 7 days) → Redash weekly dashboards`
- **Перед расширением гипотезы/ICP:** `Enrichment → прогон ключевых агентов по сегменту → обновление гипотез`
